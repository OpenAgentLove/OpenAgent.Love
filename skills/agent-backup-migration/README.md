   • 难度：⭐⭐ 中等

   • 适用：长期备份、灾难恢复
   • 速度：取决于网络
   • 难度：⭐⭐ 中等

### 3. 方案详细配置

#### 方案 1：本地复制
- 配置项：源机器人、目标机器人
- 格式：`源机器人，目标机器人`
- 示例：`小明，小红`

#### 方案 2：SSH 克隆
- 配置项：主机、端口、用户、认证
- 格式：`host:port:user:password`
- 示例：`192.168.1.100:22:root:password123`

#### 方案 3：云备份
- 配置项：存储位置、频率、内容
- 格式：`存储位置，频率，内容`
- 示例：`本地下载，一次性，完整备份`

### 4. 方案对比功能

**功能**: `对比` 命令

```
📊 方案详细对比

━━━━━━━━━━━━━━━━━━━━

| 维度 | 本地复制 | SSH 克隆 | 云备份 |
|------|----------|--------|--------|
| 速度 | ⚡ 秒级 | ⏱️ 分钟级 | 🌐 取决于网络 |
| 难度 | ⭐ 最简单 | ⭐⭐ 中等 | ⭐⭐ 中等 |
| 适用 | 同服务器 | 跨服务器 | 长期备份 |
| SSH | ❌ 不需要 | ✅ 需要 | ❌ 不需要 |
| 自动化 | ✅ 全自动 | ✅ 全自动 | ⚠️ 半自动 |

推荐建议：
• 同一服务器 → 选【1】本地复制
• 有新服务器 SSH → 选【2】SSH 克隆
• 只要备份数据 → 选【3】云备份
```

## 🔧 技术实现

### 状态管理
- 使用 `Map` 存储用户状态（内存版）
- 支持多用户并发使用
- 状态包含：当前步骤、选择的方案、配置参数、历史记录

### 步骤流程
```
步骤 0: 功能介绍 → 用户回复"开始"
步骤 1: 方案选择 → 用户选择 1/2/3
步骤 2: 方案配置 → 用户提供配置参数
步骤 3: 执行确认 → 用户确认开始
步骤 4: 执行中 → 自动执行迁移
步骤 5: 完成 → 显示完成消息
```

### 输入处理
- 支持中英文逗号分割
- 支持多种格式的命令（开始/1/方案 1 等）
- 智能识别用户意图
