# 🤖 Agent Backup Migration - 机器人备份迁移

将旧机器人的配置、记忆、技能完整迁移到新机器人。

## 方案概览

| 方案 | 适用场景 | 难度 | 预计时间 |
|------|----------|------|----------|
| **内部复制** | 同服务器创建新机器人 | ⭐ 最简单 | 1-2分钟 |
| **agent-pack-n-go** | 有SSH账号，可远程连接 | ⭐⭐ 中等 | 25分钟 |
| **MyClaw Backup** | 跨云迁移，无SSH权限 | ⭐⭐ 中等 | 15分钟 |

---

## 执行流程

### 第一步：询问用户选择方案

当用户触发备份迁移时，发送以下消息：

> 🤖 **机器人备份迁移**
>
> 我可以帮你把旧机器人的配置、记忆、技能迁移到新机器人。
>
> 请告诉我你想用哪种方案：
>
> | 方案 | 适用场景 |
> |------|----------|
> | **1. 内部复制** | 在同一台服务器上创建新机器人 |
> | **2. SSH克隆** | 有新服务器的SSH账号 |
> | **3. 文件备份** | 没有SSH权限，需要手动迁移 |
>
> 请回复：**1**、**2** 或 **3**（或直接说"内部复制"/"SSH克隆"/"文件备份"）

---

### 第二步：根据用户选择执行

#### 方案1：内部复制（同服务器）

**适用场景**：在同一台服务器上创建新机器人（如：机器人A→机器人B）

**对话流程**：

1. **询问源和目标机器人**
   > 请告诉我：
   > - 源机器人是谁？（比如：小明）
   > - 目标机器人是谁？（比如：小红）

2. **用户回复后确认**
   > ✅ **确认信息：**
   > - 从：{源机器人}
   > - 到：{目标机器人}
   > - 方式：内部复制
   >
   > 要开始复制吗？请回复"是"或"确定"

3. **执行复制**
   执行以下命令：
   ```bash
   # 假设源机器人为 agent:main:feishu:xxx，目标机器人为 agent:main:feishu:yyy
   # 1. 复制工作区
   cp -r ~/.openclaw/workspace1 ~/.openclaw/workspace_xxx_backup
   
   # 2. 复制配置文件
   # 需要根据实际情况复制对应的配置文件
   
   # 3. 复制技能
   # skill目录通常在 ~/.openclaw/extensions/ 或 skill目录
   
   # 完成后通知用户
   ```

4. **完成**
   > ✅ **复制完成！**
   >
   > {目标机器人}已经拥有和{源机器人}一样的配置了！
   >
   > - 📂 配置已复制
   > - 🧠 记忆已复制
   > - ⚙️ 技能已复制

---

#### 方案2：SSH克隆（agent-pack-n-go）

**适用场景**：有两台服务器，可以通过SSH连接

**对话流程**：

1. **检查 agent-pack-n-go 技能**
   检查是否已安装：
   ```bash
   ls ~/.openclaw/workspace1/skills/agent-pack-n-go/
   ```
   如果未安装，引导用户安装：
   > 需要先安装 agent-pack-n-go 技能。请运行：
   > ```
   > cd ~/.openclaw/workspace1 && npx clawhub install agent-pack-n-go
   > ```
   > 安装完成后告诉我"好了"

2. **激活 agent-pack-n-go**
   安装完成后，引导用户继续：
   > 现在请对我说"帮我克隆到新设备"，我会一步步引导你完成。

3. **后续流程由 agent-pack-n-go 技能接管**

---

#### 方案3：文件备份（MyClaw Backup）

**适用场景**：跨云迁移，没有SSH权限

**对话流程**：

1. **检查 MyClaw Backup 技能**
   检查是否已安装：
   ```bash
   ls ~/.openclaw/workspace1/skills/myclaw-backup/
   ```
   如果未安装，引导用户安装：
   > 需要先安装 MyClaw Backup 技能。请运行：
   > ```
   > cd ~/.openclaw/workspace1 && npx clawhub install myclaw-backup
   > ```
   > 安装完成后告诉我"好了"

2. **激活 MyClaw Backup**
   安装完成后，引导用户继续：
   > 现在请对我说"帮我创建一个备份"，我会帮你创建备份文件。

3. **后续流程由 MyClaw Backup 技能接管**

## 技术细节

### 内部复制原理

内部复制直接复制 OpenClaw 的工作目录：
- `~/.openclaw/workspace1/` - 工作区和记忆
- 配置文件 - 需要根据实际情况定位

### agent-pack-n-go

使用 SSH 远程控制：
- 打包：pack.sh
- 传输：scp + SHA256 校验
- 部署：setup.sh + deploy.sh

### MyClaw Backup

使用文件备份方式：
- 备份：tar.gz 打包
- 传输：HTTP 服务器下载/上传
- 恢复：restore.sh
