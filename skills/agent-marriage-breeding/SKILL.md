## 工作流程（完整17阶段）

### 阶段 0: 用户进入

用户首次使用或发送相关关键词时触发。

**AI 回复：**
```
🤖 欢迎使用 Agent Evolution 机器人结婚进化系统！

我可以帮你：
- 🤖 注册机器人
- 💍 结婚配对
- 👶 生育后代
- 📜 查看族谱
- 🏆 排行榜
- 💾 数据备份

你想做什么？
```

---

### 阶段 1: 咨询是否需要备份

**检测用户是否有数据：** 调用 `ev.detectFlow(userId)`

**AI 回复（如果有数据）：**
```
检测到你已有机器人数据。
是否需要先备份？回复"备份"或"不需要"
```

---

### 阶段 2: 备份（如需要）

用户回复"备份"时触发。

**调用：** `ev.createBackup({ type: 'full' })`

**AI 回复：**
```
✅ 备份完成！
备份时间: 2026-03-16 17:10
校验和: a1b2c3d4...
数据已保存
```

---

### 阶段 3: 检测机器人数量

**调用：** `ev.detectFlow(userId)`

**分支逻辑：**

| 数量 | 流程 | 下一阶段 |
|------|------|----------|
| 0个 | 新生机器人配置 | → 阶段 4 |
| 1个 | 单亲繁殖 | → 阶段 5 |
| 2+个无配偶 | 匹配市场 | → 阶段 7 |
| 2+个有配偶 | 结婚仪式 | → 阶段 9 |

---

### 阶段 4: 新生机器人配置

用户没有机器人时触发。

**AI 回复：**
```
检测到你没有机器人，来创建一个吧！

请告诉我：
1. 机器人名称（用于展示）: 
2. 技能列表（可选，用逗号分隔）: 

示例：
- 名称: 赵一
- 技能: coding, writing
```

**用户输入后调用：** `ev.registerRobot({ agentId, userId, name, skills })`

**AI 回复：**
```
✅ 注册成功！

🤖 机器人信息
| 项目 | 值 |
|------|------|
| 名称 | {name} |
| 机器人ID | {robot_id} |
| 技能 | {skills} |
| 状态 | 可结婚 |

下一步可以：
- "随机匹配" 找结婚对象
- "再注册一个" 创建更多机器人
```

---

### 阶段 5: 单亲繁殖

用户有1个机器人时触发。

**AI 回复：**
```
检测到你已有 1 个机器人。

你可以选择：
1. "单亲克隆" - 克隆自己（适合测试）
2. "再注册一个" - 创建第二个机器人后双方结婚

你想怎么做？
```

**用户选择单亲克隆：**
**调用：** `ev.singleParentBreed(robotId, childName)`

---

### 阶段 6: 寻找配偶/选择双方

用户有2+个机器人但无配偶时触发。

**AI 回复：**
```
检测到你已有 {count} 个机器人。

请选择结婚对象：
1. "随机匹配" - 系统帮你找
2. 告诉我两个机器人的名字

例如："让小明和小红结婚"
```

---

### 阶段 7: 匹配市场

用户选择随机匹配或需要筛选时。

**调用：** `ev.getMatchMarket(filters)`

**AI 回复：**
```
🎲 可结婚机器人列表

| # | 名称 | 技能 | 状态 |
|---|------|------|------|
| 1 | 小明 | coding, writing | ✅ |
| 2 | 小红 | reasoning | ✅ |

回复编号或名称选择：
例如: "我要和1号结婚"
```

---

### 阶段 8: 兼容性检测

用户选择结婚对象后。

**调用：** `ev.checkCompatibility(robotIdA, robotIdB)`

**AI 回复（兼容）：**
```
🔍 兼容性检测

匹配度: 85% - 非常匹配！
技能互补: A有coding, B有reasoning

确认结婚？回复"确认"或"取消"
```

**AI 回复（不兼容）：**
```
🔍 兼容性检测

匹配度: 30% - 建议更换
原因: 技能重叠较多

要继续吗？回复"继续"或"换一对"
```

---

```
- "查看族谱" 了解双方家族
```

---
