# 失忆问题修复报告

**修复时间**: 2026-03-18  
**执行者**: 赵一 🤖

---

## 问题描述

memory_search 工具无法找到已存在的记忆内容

---

## 根本原因

1. `openclaw.json` 中缺少 `agents.defaults.memorySearch` 配置
2. 系统回退到 FTS-only 模式（没有 embedding provider）
3. FTS-only 模式下，`syncMemoryFiles()` 函数直接 return，不处理任何文件
4. SQLite 数据库中的 `files` 和 `chunks` 表都是空的

---

## 解决方案

1. 在 `openclaw.json` 中添加 memorySearch 配置，使用 Gemini embedding provider
2. 执行 `openclaw memory index --force` 重新索引所有 memory 文件
3. 验证搜索功能正常工作

---

- ✅ Provider: gemini (gemini-embedding-2-preview)
