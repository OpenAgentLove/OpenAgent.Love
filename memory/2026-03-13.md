## 2026-03-13 更新：Gateway 网关文档

### 核心概念
- **Gateway**: OpenClaw 的 WebSocket 网关服务，负责路由、控制平面、渠道连接
- 默认端口: **18789**
- 默认绑定: loopback
- 认证: 默认开启 Token 认证

### 常用命令
```bash
openclaw gateway status          # 检查状态
openclaw gateway start           # 启动
openclaw gateway stop            # 停止
openclaw gateway restart         # 重启
openclaw gateway install         # 安装系统服务
openclaw logs --follow           # 查看日志
openclaw health                  # 健康检查
openclaw status                  # 完整状态
```

### 热重载模式
| 模式 | 行为 |
|------|------|
| off | 无重载 |
| hot | 仅热安全变更 |
| restart | 需重启的变更 |
| hybrid | 自动选择（默认）|

### 故障排查
| 签名 | 问题 |
|------|------|
| "refusing to bind without auth" | 非 loopback 绑定但无认证 |
| "EADDRINUSE" | 端口冲突 |
| "unauthorized" | 客户端/网关认证不匹配 |

## 2026-03-13 更新：新手引导向导

### 入口命令
```bash
openclaw onboard       # 交互式向导
openclaw configure     # 重新配置
openclaw dashboard     # 打开控制界面
```

### 快速开始 vs 高级
- **快速开始**: 保持默认（本地 Gateway、端口 18789、Token 认证）
- **高级**: 完全控制每一步

### 向导流程
1. 检测现有配置
2. 模型/认证（Anthropic/OpenAI/MiniMax/Moonshot 等）
3. 工作区设置
4. Gateway 配置
5. 渠道配置（Telegram/WhatsApp/Discord/Signal 等）
6. 守护进程安装（systemd/launchd）
7. 健康检查
8. Skills 安装

### 非交互模式
```bash
openclaw onboard --non-interactive \
  --mode local \
  --auth-choice apiKey \
  --anthropic-api-key "$KEY" \
  --gateway-port 18789 \
  --install-daemon
```
