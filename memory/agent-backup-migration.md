## 三种方案汇总

### 方案1：内部直接复制（同服务器/同机器）
- 适用场景：机器人A→机器人B，在同一台服务器上
- 特点：最简单，直接复制文件
- 代表工具：无（手动操作）

### 方案2：agent-pack-n-go（本地SSH克隆）
- 适用场景：本地→本地，可以SSH连接
- 特点：纯SSH传输，零第三方依赖
- 仓库：https://github.com/aicodelion/agent-pack-n-go

### 方案3：MyClaw Backup（云端文件备份）
- 适用场景：跨云迁移、无SSH权限
- 特点：生成备份文件，通过HTTP导出
- 仓库：https://github.com/LeoYeAI/openclaw-backup

```
        ┌───────────┴───────────┐                             ┌───────────┴───────────┐
        │                       │                             │                       │
        ▼                       ▼                             ▼                       ▼
   【阶段1】               【阶段2】                     【阶段3】               【阶段4】
   飞行前检查             打包&传输                     远程部署                验证&切换
        │                       │                             │                       │

   ②验证SSH连接           ②transfer.sh传输             ②deploy.sh              ②切换设备
   ③设置无密码sudo        (SHA256校验)                  ③启动Gateway            ③测试消息
   ④网络诊断                                         ④验证运行                ④测试记忆
                                                    ⑤清理临时文件              ⑤庆祝🎉
```

### 详细步骤

#### 阶段1：飞行前检查（旧设备）
1. **询问新设备信息**
   - 新电脑的IP地址是多少？
   - SSH用户名是什么？（如root、ubuntu）
2. **要求用户配置SSH密钥**
   - 运行：`ssh-copy-id 用户名@新IP`
   - 这是为了后续无需密码SSH连接
3. **验证SSH连接**
   - 测试：`ssh 用户名@IP 'echo ok'`
4. **设置无密码sudo**（可选但推荐）
   - 运行：`ssh 用户名@新IP 'echo "用户名 ALL=(ALL) NOPASSWD:ALL" | sudo tee /etc/sudoers.d/migration'`
5. **网络诊断**
   - 运行网络检查脚本，判断直连/代理/无网络

#### 阶段2：打包&传输（旧设备）
1. **打包**：`pack.sh` 打包配置、记忆、技能、凭证
2. **传输**：`transfer.sh 用户名@新IP` 通过scp传输
3. **校验**：SHA256校验确保完整性

#### 阶段3：远程部署（新设备）
1. **基础环境安装**：`setup.sh` 安装nvm、Node.js 22、Claude Code
2. **OpenClaw部署**：`deploy.sh` 部署OpenClaw、恢复配置
3. **启动验证**：`openclaw gateway status` 确认运行
4. **清理临时文件**

#### 阶段4：验证&切换
1. **检查日志**：查看Discord/飞书连接状态
2. **设备切换**：
   - 旧设备执行：`openclaw gateway stop`
   - 新设备确认运行
3. **测试**：
   - 消息测试：发消息看是否回复
   - 记忆测试：问"你还记得我是谁吗"
4. **庆祝**：克隆成功！

## 方案3：MyClaw Backup 详细流程

### 树状结构

```
                         ┌─────────────────────────────────────────────┐
                         │         MyClaw Backup 完整流程              │
                         └─────────────────────────────────────────────┘
                                            │
                    ┌───────────────────────┴───────────────────────┐
                    │                                                   │
                    ▼                                                   ▼
        ┌───────────────────────┐                         ┌───────────────────────┐
        │    旧设备（备份端）      │                         │    新设备（恢复端）      │
        └───────────────────────┘                         └───────────────────────┘
                    │                                                   │
        ┌───────────┴───────────┐                             ┌───────────┴───────────┐
        │                       │                             │                       │
        ▼                       ▼                             ▼                       ▼
   【备份流程】              【下载流程】                    【恢复流程】              【验证流程】
        │                       │                             │                       │
   ①backup.sh              ①启动serve.sh                 ①安装OpenClaw             ①检查运行
   ②创建tar.gz             ②浏览器打开URL                ②安装备份技能              ②检查日志
   ③启动HTTP服务          ③下载文件                     ③curl下载                  ③测试对话
   ④通知URL+token         ④关闭HTTP                    ④dry-run预览              ④确认完成
                                                   ⑤确认恢复
                                                   ⑥restore.sh
```

### 详细步骤

#### 备份端（旧设备）
1. **创建备份**：`bash scripts/backup.sh /tmp/openclaw-backups`
2. **启动HTTP服务**：`bash scripts/serve.sh start --token TOKEN --port 7373`
3. **提供下载信息**：告诉用户URL和token
4. **等待用户下载完成**
5. **关闭HTTP服务**：`bash scripts/serve.sh stop`

#### 下载端（用户操作）
1. **浏览器打开URL**：http://IP:7373/?token=TOKEN
2. **点击下载按钮**
3. **保存到本地**

#### 恢复端（新设备）
1. **安装OpenClaw**（如没有）
2. **安装备份技能**：`clawhub install myclaw-backup`
3. **下载备份文件**：`curl -O "http://旧IP:7373/download/xxx.tar.gz?token=TOKEN"`
4. **预览恢复**：`bash scripts/restore.sh xxx.tar.gz --dry-run`
5. **确认恢复**：回答"是"或"确定"
6. **执行恢复**：`bash scripts/restore.sh xxx.tar.gz`

#### 验证流程
1. **检查运行**：`openclaw gateway status`
2. **检查日志**：查看连接状态
3. **测试对话**：发消息测试
